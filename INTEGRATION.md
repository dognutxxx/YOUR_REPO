# การนำโปรเจกต์นี้ไปใช้ในโปรเจกต์อื่น (Integration Guide)

> **สถานะ:** โปรเจกต์นี้ยังพัฒนาไม่เสร็จ (work in progress)

เมื่อคุณนำไฟล์จากโปรเจกต์ **3D Particle Morphing on Scroll** ไปใส่ในอีกโปรเจกต์หนึ่ง โปรเจกต์ปลายทางจะได้ทั้ง **Frontend (FE)** และ **Backend (BE)** ไม่ได้โดยตรง — แต่จะได้ **ดีไซน์และสไตล์ของโปรเจกต์นี้** (ฟอนต์ สี พื้นหลัง ระยะเลื่อน ฯลฯ) แทนดีไซน์เดิมที่เรียบง่ายของโปรเจกต์ปลายทาง

เอกสารนี้ใช้สำหรับ:
- นำไฟล์ที่เกี่ยวข้องไปวางในอีกโปรเจกต์
- ให้ AI/Agent แก้ฟอนต์และดีไซน์ให้ใกล้เคียงกับโปรเจกต์นี้ได้
- อ้างอิงค่าสี ฟอนต์ และโครงสร้าง CSS

---

## ไฟล์ที่เกี่ยวข้อง (นำไปใช้ในโปรเจกต์อื่น)

| ไฟล์/โฟลเดอร์ | คำอธิบาย |
|---------------|----------|
| `index.html` | โครงหน้า + สไตล์ (font, color, layout) |
| `js/main.js` | โลจิก 3D, particle, shader, scroll |
| โฟลเดอร์ `js/` | ต้องมี `main.js` |

**CDN ที่ใช้ใน `index.html`:** Three.js, GSAP, ScrollTrigger, Lenis (ใน `main.js` เป็น ESM)

---

## ดีไซน์เริ่มต้นของโปรเจกต์นี้ (Default Design)

เมื่อนำไฟล์นี้ไปใช้ โปรเจกต์ปลายทางจะเปลี่ยนไปใช้ดีไซน์ชุดนี้:

### ฟอนต์ (Font)

- **Font family:** `system-ui, -apple-system, sans-serif`
- **หัวข้อ (h1):** `font-size: clamp(2rem, 5vw, 4rem)`, `font-weight: 700`
- **ข้อความ (p):** ความกว้างประมาณ `40ch`, `margin-top: 1rem`, `opacity: 0.85`

### สี (Colors)

- **พื้นหลัง (body):** `#0a0a0f`
- **ข้อความหลัก:** `#e0e0e0`
- **เงาหัวข้อ:** `0 0 40px rgba(100, 200, 255, 0.3)` (โทนฟ้าอ่อน)

### Layout

- **Canvas:** เต็มจอ, `position: fixed`, `z-index: 0`
- **เนื้อหาเลื่อน:** `min-height: 400vh`, `z-index: 1`
- **Section:** `min-height: 100vh`, flex กลางจอ, `padding: 2rem`
- **Intro section:** `padding-top: 20vh`
- **Outro section:** `padding-bottom: 20vh`

### อื่นๆ

- **Scroll:** `scroll-behavior: auto` (ให้ Lenis ควบคุม)
- **Box model:** `* { margin: 0; padding: 0; box-sizing: border-box; }`

## การให้ AI/Agent แก้ฟอนต์และดีไซน์ให้เหมือนโปรเจกต์นี้

ถ้าอยู่ในอีกโปรเจกต์และต้องการให้ฟอนต์/ดีไซน์คล้ายโปรเจกต์นี้ ให้:

1. **อ้างอิงไฟล์นี้:** ใส่ไฟล์ `INTEGRATION.md` นี้ (หรือส่วนที่เกี่ยวกับดีไซน์) ไว้ในโปรเจกต์ปลายทาง แล้วบอก Agent ว่า “ให้ใช้ดีไซน์ตาม INTEGRATION.md”
2. **หรือคัดลอกค่าด้านบน:** ส่งส่วน **ดีไซน์เริ่มต้นของโปรเจกต์นี้** ให้ Agent แล้วบอกว่า “ให้ฟอนต์และสีเป็นแบบนี้”
3. **หรืออ้าง path เดิม:** ถ้ายังเก็บ `index.html` ของโปรเจกต์นี้ไว้ ให้บอก Agent ว่า “ให้ฟอนต์และ layout ตาม design ใน [path ไปที่ index.html]”

Agent จะใช้ค่าฟอนต์ สี และ layout ข้างบนเป็น reference เพื่อแก้ไฟล์ในโปรเจกต์ปลายทางให้ใกล้เคียงกับโปรเจกต์นี้ได้

---

## สรุป

- **นำไฟล์ไปอีกโปรเจกต์:** ใช้ `index.html`, `js/main.js` (และโฟลเดอร์ `js/`) ตามตารางด้านบน
- **ดีไซน์เดิมของโปรเจกต์ปลายทาง:** จะถูกแทนที่ด้วยดีไซน์ของโปรเจกต์นี้ (ฟอนต์ สี พื้นหลัง ระยะเลื่อน) ตามค่าที่เขียนในส่วน “ดีไซน์เริ่มต้น”
- **การแก้ฟอนต์/ดีไซน์ในโปรเจกต์อื่น:** ใช้ `INTEGRATION.md` นี้เป็น reference หรือให้ Agent อ่านไฟล์นี้แล้วปรับให้ตรงกับดีไซน์นี้

*โปรเจกต์นี้ยังพัฒนาไม่เสร็จ — โครงหรือค่าบางอย่างอาจเปลี่ยนในอนาคต*
